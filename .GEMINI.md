# Shukr Foundation Mandates

## 🧪 Testing & Code Coverage
- **Mandate:** All new code and refactors MUST maintain a minimum of **90% code coverage**.
- **Rule Enforcement:** Future AI agents and developers must verify coverage before considering a task complete.
- **Verification:** Run the project's test suite with coverage reporting enabled (e.g., `npm run test:coverage` or `vitest run --coverage`).

## ⚖️ Design System Integrity
- **Mandate:** All UI/UX development, styling, and component creation MUST strictly follow the specifications in `design-system/MASTER.md`.
- **Consistency:** Maintain the "Comfort & Empathy" aesthetic (Deep Sage, Soft Gold, Warm Cream) across all screens.
- **Accessibility:** Prioritize high-legibility Nastaliq Urdu and large touch targets for senior users.
- **Haptic Feedback:** All primary interactions (taps, long-press) should provide subtle vibration feedback if supported.

## 📱 Mobile-First Layout
- **Rule:** Optimize specifically for high-aspect-ratio Samsung/Android devices.
- **Rule:** Use `100dvh` for all viewport heights to prevent vertical overflow.
- **Rule:** Maintain the Instagram-inspired "Story & Feed" pattern for familiar navigation.

## 📦 Data Management
- **Boot Data:** `public/boot_data.json` is the source of truth.
- **Portability:** Use `UniversePorter` for all data migration and synchronization tasks.

## 🛠️ Technical Quality
- **Standard:** Follow clean-code principles. No "slop" or redundant styling.
- **Standard:** Always use Playwright to verify UI integrity before final delivery.
